Demonstration of Generating a Bottleneck Steiner Tree in Euclidean Plane
  • Admin
  • Creative
  • 0 Comments

deployed on Github Pages:

The bottleneck k-Steiner tree problem is described as follows: As n points and a positive integer k are given, it’s required to find a Steiner tree that joins n points with at most k Steiner points for minimizing the longest edge of this Steiner tree. L. Wang and D.-Z. Du [1] demonstrate an approximate performance ratio of 2 for bottleneck Steiner tree problem in Euclidean plane. And an algorithm for the problem with time complexity of O(n𝑙𝑜𝑔 𝑛+kn) is given. In this study, a Web application, called GBST, has been created for simulating the problem and demonstrating the algorithm, and the algorithm is optimized by introducing the Heap. The time complexity of the optimized algorithm is O(n𝑙𝑜𝑔 𝑛+k𝑙𝑜𝑔 𝑛), and the optimized algorithm can be applied more efficiently. Additionally, the implementation of the GBST can be directly applied in assistance teaching of undergraduate and postgraduate students.

In the design of wireless communication networks, with the limit of the project’s budget, we suppose that the n local base stations are reused and the k new stations plan to be built in the area. However, n local base stations are located at the given points in the plane, and their distances between them have a trouble with the quality of communication because of signals’ missing with a long distance. Other k stations are aiming for improving the quality of communication by adding to the plane. How to choose locations for other k stations to minimize the longest distance between stations is the problem.

This problem can be descripted as the following bottleneck Steiner tree problem: Given a set P = {𝑝1, 𝑝2,..., 𝑝n} of n terminals and a positive integer k, the goal is to find a Steiner tree with at most k Steiner points such that the length of the longest edges in the tree is minimized.

However, the conventional Steiner tree problem has been proved to be NP-hard [2] which drives many scientists to find an optimal approximation. Then, a polynomial-time approximation scheme (PTAS) [3] for the conventional Steiner tree problem in the Euclidean plane is introduced by Arora, which may be the best one.

Covering the areas of VLSI routing, wireless communication and Evolutionary Trees, the Steiner tree problem has been deeply discovered and researched, which often require some modifications to the classic Steiner tree problem with requiring new techniques. Therefore, the study of the variant of the Steiner tree has become a hot technology topic in recent years.

L. Wang and D.-Z. Du demonstrate an approximate performance ratio of 2 for the Euclidean plane bottleneck Steiner tree algorithm. And an algorithm for the problem with time complexity of O(n𝑙𝑜𝑔 𝑛+kn ) is given. However, the algorithm can be optimized by introducing the Heap. The time complexity of the optimized algorithm is O(n𝑙𝑜𝑔 𝑛+k𝑙𝑜𝑔 𝑛), and the optimized algorithm can be applied more efficiently. The algorithm is decomposed into three steps:

  • The n points are put on the plane by their co-ordinates and Delaunay triangulation algorithm is applied for pre-processing them.
  • Kruskal algorithm and Prim algorithm are respectively applied to find the Minimum Spanning Tree of that with different Time Complexity. Additionally, Prim algorithm is optimized by Minimum Heap.
  • k Steiner points are put on the edge with largest value after adding all edges to a Maximum Heap.

Additionally, in each step, its algorithms are analyzed and compared by Time Complexity and their time-consuming curves. So, the algorithm gets optimized.
